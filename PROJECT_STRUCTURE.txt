Key Directories Explained
==========================

Backend (Python)
----------------
- backend_api.py        : FastAPI server with REST endpoints
- sending_update.py     : Core email processing logic
- receive_email.py      : Email inbox monitoring
- HCN1.xlsx            : Excel database with booking data

Frontend (React)
----------------
Components:
  - Header.tsx          : Top navigation bar
  - StatCard.tsx        : Dashboard statistics cards
  - BookingTable.tsx    : Reusable table for displaying bookings
  - ProcessButton.tsx   : Button to trigger email workflow

Pages:
  - Dashboard.tsx       : Overview with statistics
  - AllBookings.tsx     : All bookings list
  - PendingBookings.tsx : Bookings awaiting HCN
  - CriticalBookings.tsx: Bookings with issues
  - Settings.tsx        : System configuration view

Services:
  - api.ts             : Axios HTTP client for backend communication

Hooks:
  - useBookings.ts     : React Query hooks for data fetching

Types:
  - index.ts           : TypeScript type definitions


Technology Stack
================

Backend:
  - Python 3.8+
  - FastAPI (REST API)
  - Pandas (Excel processing)
  - OpenAI (Email analysis)
  - Gmail SMTP/IMAP

Frontend:
  - React 18
  - TypeScript
  - Vite (Build tool)
  - Tailwind CSS
  - React Router v6
  - TanStack Query (React Query)
  - Axios


API Endpoints
=============

GET  /                          Health check
GET  /api/status                Get booking statistics
GET  /api/bookings              Get all bookings
GET  /api/bookings/pending      Get pending HCN bookings
GET  /api/bookings/critical     Get critical bookings
GET  /api/config                Get system configuration
POST /api/process               Start email processing workflow


Running the Application
=======================

Quick Start:
  ./start.sh

Or manually:

  Terminal 1 (Backend):
    cd /Users/amarjeet/Downloads/hcn
    source .venv/bin/activate
    python backend_api.py

  Terminal 2 (Frontend):
    cd /Users/amarjeet/Downloads/hcn/frontend
    npm run dev

Access:
  Frontend:  http://localhost:3000
  Backend:   http://localhost:8000
  API Docs:  http://localhost:8000/docs
